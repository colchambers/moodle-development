Client Interface

A demonstration version of the Offline Moodle Administration Centre has been created to provide a clear indication of the functionality we intend to provide. The currnet version has been agreed with The Open University in principle and will be developed over the next coming versions with further further functionality developed.

For the purposes of this demonstration we have focused on developing the following core functionality which will be available to all pages within the administration centre:
  • the intended look and feel.
  • the intended navigational elements.
  • examples of key pages and sections including; login, home, course details.
We believe that that there are two general types of screen the application will require:
  1. Screens to enter and manage information such as presentations and documents.
  2. Screens to view information in a such as a workflow format, forum etc.
With this release we have provided examples of screens to input and manage document information. These screens will form the basis of all other screens where information is input. We have not provided an example of how the workflow screens will look, this will be provided in a future release.

Starting point

The demonstration begins in the administration centre viewing the login screen.

Procedure

The demonstration screens are static and therefore do not expect any login details. Therefore click the 'Login' button to reach the home page and begin browsing the administration centre.

Presentation Details

The presentation details section has been developed first, since it will form a template of how to enter and manage information for the rest of the administration centre. We are currently working through each part of the section and defining how we will provide the required functionality.

The following list indicates the order in which we are currently developing the functionality and which areas are complete and ready for demonstration:
  1. Site home page
  2. Synchronisation
    1. Home page
    2. Configuration
    3. Synchronise
      1. Select Items
      2. Resolve Conflicts
      3. Display Results
  3. Course
    1. Home page
    2. Synchronisation Filtered
    3. Synchronised
  4. Forum
    1. Home page
    2. Synchronisation Filtered
    3. Synchronised

To date we have developed the functionality and look of the resolve conflicts section. We have created a 'Resolve Conflicts' page which allows items that cannot be synchronised automitically to be reviewed and decisions made. This method is aimed addressing conflicts that the system cannot resolve itself.

The functionality of all pages within all other sections will use the resolve conflicts section as a template. Therefore the input and feedback that The Open University provide regarding this is imperative.
