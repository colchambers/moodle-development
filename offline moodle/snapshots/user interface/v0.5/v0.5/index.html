Version 0.5: Introduction

Web development is a process of turning the clients requests into reality. There is considerable room for interpretation of customer requirements and therefore we aim to provide our clients methods whereby they can sample our interpretation and interact with it as much as possible.

This demonstration package has been provided as an example of phase one of the Offline Moodle System we aim to provide. The pages within the demonstration package have been designed with sufficient functionality to provide a demonstration of key deliverables and to serve as a clear example of what has been agreed. You will therefore find that some functionality such as menus and links do not work as expected or at all. You may also find that lists of documents commonly link to the same document to ease administration of the demonstration. If you would like to discuss the functionality that will be provided in the final version then please contact us.

Phase one of the Offline Moodle System consists of one main section:
  • Offline Moodle: Client Synchronisation Interface
A demonstration of only the synchronisation interface has been included in this release. As well as providing a demonstration of the interface, we have provided short explanations of some of the key features. You can access these explanations directly using the links below. Alternatively access to all content can be achieved through the menus on the right hand side of this page. The links will open up in this browser window.
  • Synchronisation Interface

Updates In this Version

The previous snaphost v0.4 introduced some small improvements. This release reorganises the interface in an attempt to simplify the process. The following changes have been introduced:

  • Process broken into several pages: Rather than manage the entire process from one page. There will now be several pages each focusing on a different task including .
    • Find Content: Navigate the Moodle content finding items to syncronise.
    • 3 Step synchronisation: Synchronisation now has a start indicating the item to be synchronised, middle where conflicts are resolved and an end summarising what has occurred.
    • Shortcuts: For quick synchronisation of common items shortcuts can be recorded.
