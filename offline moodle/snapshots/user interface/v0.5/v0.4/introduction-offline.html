Project Specification

This document contains definitions for all key aspects of the project. In combination with this snapshot it will be used at the final review meeting as a guideline to determine if we have met the requirements set out by The Open University and to ultimately sign the project off. The specification, like the snapshot, is expected to change over time as the project develops or if requirements change. Once all parties agree on the content of the specification a copy must be agreed by a member of each party to form a binding agreement.

Document: Moodle Offline Project.V1.1.doc

Overall Software Components

To explain the source of the underlying components of the synchronisation process we created a diagram to make this easier to understand. The components of the diagram are labelled according to this key:

  • Synchronisation: a specific component will be developed for the project.
  • Web Service: Will use mNet functionality to provide a web service layer. This layer will be built for the project
  • User Interface: A user interface will be developed to provide the user with means to manage the synchronisation process
  • mNet: the Moodle Network component of Moodle will be used
  • Backup/Restore: the Backup/Restore component of Moodle will be used.
